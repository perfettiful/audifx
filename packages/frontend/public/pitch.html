AudiFX

Audio effects + AI stem separation + MIDI transcription + 3D visualization.
Looking for a backend engineer to own the audio-to-MIDI pipeline.

Overview
Two modes. Audio Effects runs in the browser: upload a song, apply effects (Slowed + Reverb, Nightcore, Lo-fi, 8D Audio, Vaporwave, Underwater, Chopped & Screwed), tweak parameters, export. Uses Tone.js and the Web Audio API. No backend needed. Deployed to GitHub Pages.

MIDI Visualizer is the backend piece. Upload audio, the server separates it into stems with Demucs (Meta), transcribes each to MIDI with BasicPitch (Spotify), cleans up the notes, and outputs a timeline.json. The frontend renders it as an interactive 3D scene in Three.js with playback sync, voice filtering, and camera controls. Also includes a stem mixer and music analysis (key, chords, PCA, Tonnetz).

What's Built
AI Stem Separation
Demucs htdemucs (Meta) splits audio into 4 stems: vocals, bass, drums, other. GPU accelerated via CUDA. Each stem saved as WAV and served via the API. Users can download or stream individual stems.
MIDI Transcription
BasicPitch (Spotify) outputs MIDI note events with onset, duration, pitch, velocity. Drums handled separately via onset detection. Post-processing removes noise, merges duplicates, clamps pitch ranges, limits polyphony. MIDI files downloadable per stem.
3D Visualization
Three.js renders all notes as 3D boxes. X = time, Y = pitch, Z = voice. Color per instrument. Notes light up during playback. Orbit and follow camera modes. Per-voice filtering. Section markers (verse, chorus). Instanced mesh rendering for performance.
Audio Effects (browser only)
7 effects with Tone.js: Slowed+Reverb (convolver), Nightcore (EQ boost), Lo-fi (band-pass + distortion + bit crush + vinyl noise), 8D (LFO panner + reverb), Underwater (lowpass), Chopped & Screwed (scheduled stutters), Vaporwave (chorus + reverb). 8 genre presets with parameter values.
Stem Mixer
Per-stem volume, pan, mute, solo controls. All stems play in sync. Remix endpoint mixes server-side with numpy, normalizes, returns WAV or FLAC.
Music Analysis
Key detection (Krumhansl-Schmuckler), chord analysis (time-windowed triad/seventh detection), PCA voice comparison (feature extraction + projection), Tonnetz torus (neo-Riemannian pitch class mapping).
REST API
14 endpoints via FastAPI (Swagger at /docs): upload, job status, timeline, audio, stems list, stem download, MIDI list, MIDI download, remix, analysis, harmony, PCA, Tonnetz, export. Jobs run in a background thread with progress.
Gaps
  • Transcription accuracy on complex passages. BasicPitch alone misses notes in dense polyphony (keys, guitar chords).
  • Drum classification is rule-based. Spectral centroid can only do kick/snare/hihat. Misclassifies toms, cymbals, etc.
  • No beat grid. Notes are raw float timestamps. No quantization to musical positions.
  • No tempo detection. BPM isn't extracted. Can't do beat-aware post-processing.
  • HTTP polling for job progress. Should be WebSocket.

madmom: beat detection
pip install madmom. Detect beats, BPM, and downbeats. Build a subdivision grid. Snap all note onsets to nearest grid point. Runs on CPU. About a day of work.
MT3: better "other" stem
Google's multi-instrument transcription model. Use for the "other" stem only, where BasicPitch struggles with complex polyphony. Merge output by deduplicating notes within 50ms + same pitch. About 2 days.
Omnizart: real drum detection
pip install omnizart. Pre-trained drum model that classifies kick, snare, hihat, toms, cymbals. Replaces the spectral centroid approach. About a day.
Docker + cloud deploy
Dockerize the backend. Deploy to RunPod at ~$0.40/hr (~$0.02 per song). Add WebSocket for progress streaming. About 2 days.

Deployment
Frontend
GitHub Pages via GitHub Actions. Deploys on push to main. Effects work with no backend. Backend features show a fallback screen when the server isn't running.
Backend
Docker container on a cloud GPU. RunPod at ~$0.40/hr. A 3 minute song takes ~90 seconds, so roughly $0.02 per song. Also runs locally on any RTX 3060+ or M1/M2 Mac.
Roadmap
Now
Beat detection and quantization
Integrate madmom for BPM and beat tracking. Build a subdivision grid from detected beats. Snap all transcribed note onsets to the nearest grid point. This is the single biggest quality improvement to the MIDI output.
Next
Better transcription and drum detection
Add MT3 (Google) as a second transcription model for the "other" stem, where BasicPitch struggles with polyphonic keys and guitar. Merge outputs by deduplicating notes within 50ms and same pitch. Replace the rule-based drum classifier with Omnizart's pre-trained drum model for proper kick/snare/hihat/tom/cymbal classification.
Then
Containerize and deploy
Dockerize the backend so it's reproducible. Deploy to a cloud GPU provider (RunPod or similar) at about $0.02 per song processed. Replace HTTP polling with WebSocket for real-time progress streaming.
Later
If there's demand
MP4 video input (ffmpeg audio extract). SQLite for job persistence. Rate limiting. More analysis features. These depend on what users actually ask for.
